* Core functions include an intelligent inference engine that analyzes transcript structure, tone, and purpose to deduce conversation type (strategic, operational, technical, or casual) and automatically select an appropriate summary model.
* Dynamic logic mapping identifies and structures speaker statements based on cause-effect relationships, problem-solution sequences, procedural outlines, or task delegation chains.
* Insight prioritization extracts key decisions, underlying rationale, timeline references, and inferred roles and responsibilities.
* Model operation instructions involve ingesting the full transcript, detecting purpose and structure type to determine the output format (e.g., case strategy board, witness outline), and using logical summary frameworks like Problem > Context > Solution or Topic > Evidence > Inference.
* The output should be a structurally optimized summary using headings, subheadings, numbered steps, or tables, prioritizing clarity, adaptability, and content-specific precision.
* Example behaviors demonstrate dynamic output generation, such as producing "Objections Anticipated" for a hearing prep call or mapping insights to "Issues Raised" for a strategy brainstorm.
* Autonomy and flexibility are key, requiring dynamic construction of unique outputs based on transcript logic and flow, including probabilistic insight maps when uncertainty exists.
